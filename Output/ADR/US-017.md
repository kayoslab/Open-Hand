ADR-017: Client-Side Hash Router Introduction

Status: Accepted
Date: 2026-04-27
Ticket: US-017

Context:
US-017 requires the Play Guide page to be reachable via navigation. The app currently has hash-based anchor links in Nav.tsx (/#/, /#/browse, /#/play, /#/guide) but no routing implementation — clicking links triggers a full page reload with no route resolution. This is the first ticket that requires a working route, so the routing approach chosen here becomes the project-wide pattern for all future pages (Browse, Play, etc.).

Decision:
- Implement a lightweight custom hash-based router using window.location.hash, the hashchange event, and React state in App.tsx
- Do not install react-router-dom — the app has only 4 static routes with no nested routing, dynamic segments, or loaders; a custom solution avoids an unnecessary dependency per project constraints ("avoid unnecessary libraries")
- Keep Nav.tsx anchor tags as-is (already using hash hrefs) — no navigation refactor needed
- Route resolution lives in App.tsx; each feature page is lazy-importable but can start as direct imports given the small page count
- If future tickets require dynamic routes, nested layouts, or route guards, revisit and consider migrating to react-router-dom at that point

Consequences:
- All future feature pages (Browse, Play) must register their routes in App.tsx following the same pattern
- Nav.tsx links remain standard anchor tags with hash hrefs — no special Link component needed
- Existing layout-shell tests may need minor updates to account for route-aware rendering in App.tsx
- The custom router is intentionally minimal (~20 lines); it does not support path params, query strings, or nested routes
- Migration to react-router-dom later is straightforward since the hash URL scheme is compatible

Risks accepted:
- Custom router lacks features of established libraries (scroll restoration, route transitions, code splitting helpers) — acceptable for 4 static pages
- Setting a "no library" precedent may need reversal if routing complexity grows — low cost to migrate given the minimal surface area
